# Status Skill — Bootstrap Template

## Bootstrap: questions for the operator

When the operator wants to install `/status`, ask these questions **before**
generating SKILL.md:

```
For the /status skill I need a quick note on:

1. Which background processes/daemons run in your system?
   (Comma-separated, e.g. "run-loop.sh, api-server.js, worker.py")

2. Are there data files or signal files whose freshness is relevant?
   (e.g. "signals/*.json show whether agents deliver data")
   If no: just say "No"

3. Does the system have a dashboard or health endpoint?
   (e.g. "port 3000", "http://localhost:8080/health", or "No")

4. Which log files are most relevant for the overview?
   (e.g. "app.log", or "No / unclear yet")
```

Wait for answers. Then generate SKILL.md from the skeleton below and write it to
`{PROJECT_PATH}/.claude/skills/status/SKILL.md`.
Replace placeholders:
- `{{PROJECT_NAME}}` → from phase 0 info-gathering
- `{{DAEMON_LIST}}` → answer 1 as a bullet list
- `{{SIGNAL_FILES}}` → answer 2 (or "— (not configured)")
- `{{DASHBOARD_URL}}` → answer 3 (or "— (no dashboard)")
- `{{LOG_FILES}}` → answer 4 (or "— (not configured yet)")

## Skeleton SKILL.md (generated)

```markdown
---
name: status
version: 1.0.0
description: On-demand system status dashboard for {{PROJECT_NAME}}. Shows daemon status, data freshness, open issues, and system health in one compact view.
---

# /status — System status dashboard

**Triggers:** when the operator says "status", "/status", or "how's it going?"

Run all checks, then output a compact table.

---

## Check 1: Daemons / background processes

{{DAEMON_LIST}}

```bash
# For each daemon check whether the process is running:
# ps aux | grep "{daemon-name}" | grep -v grep | head -1

# If lock files exist:
# ls -la ./*.lock 2>/dev/null
```

Status columns: ✅ Running | ❌ Down | ⚠️  Unknown

---

## Check 2: Data freshness

Signal / data files: {{SIGNAL_FILES}}

```bash
# For every relevant file: compute age in minutes
# find signals/ -name "*.json" -newer /tmp/ref_5min 2>/dev/null
# (ref_5min = touch -t $(date -d '5 minutes ago' +%Y%m%d%H%M) /tmp/ref_5min)
```

Status columns: ✅ Fresh (<5 min) | ⚠️  Stale (5–30 min) | ❌ Old (>30 min)

---

## Check 3: Dashboard / health endpoint

Dashboard: {{DASHBOARD_URL}}

```bash
# Health check:
# curl -s -o /dev/null -w "%{http_code}" {{DASHBOARD_URL}}/health 2>/dev/null
```

---

## Check 4: Recent errors in the log

Log files: {{LOG_FILES}}

```bash
# Errors of the last hour:
# grep -i "error\|fatal\|crash" {{LOG_FILES}} | tail -5
```

---

## Check 5: Git status

```bash
cd {PROJECT_PATH}
git status --short
git log --oneline -3
```

At the end output a compact table:

|---------|--------|---------|

### Data freshness
| File / source | Age | Status |
|---------------|-----|--------|
| [file1] | 2 min | ✅ |
| [file2] | 45 min | ❌ |

### System
| Check | Status |
|-------|--------|
| Dashboard | ✅ port 3000 reachable |

### Summary
[HEALTHY / DEGRADED / CRITICAL] — [one-sentence summary]
```
